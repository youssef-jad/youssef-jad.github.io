Ah, the nightmare of selecting a blogging system — too many choices. Should I get a domain? Should I custom build it? Overthinking is killing me, and overengineering is right around the corner, turning my weekend into a nightmare. It's like being level 1 fighting Malenia in Elden Ring.

Make a coffee, think, open Google, do some searches — "yes, I still do manual searches 😄" — open Gemini, compare my manual searches with AI conversation results, check Claude, and make another coffee 😄

I know this rabbit hole, and I am Alice. I will not release my blog on the weekend if I keep running like this, obviously 😄

Cutting to the Chase

Keeping it Simple

So I decided to cut to the chase. I don't want a fancy dashboard, I don't need to pay for a domain — I just need to start. I selected Hugo and GitHub Pages.

I took one step back — all I need is a place to publish some content. I'm already active on Medium and write on daily.dev, so a nice personal blog is what I need to reflect everything on my mind, with all the UI freedom I want — not just Markdown articles.

So I started keeping things simple, put things up, made a demo page, added my first article, and here we go: https://youssef-jad.github.io/ — my blog is up! Using Antigravity I built a good UI, and I told Claude and OpenClaw to sync my articles — like magic, I'm not wandering around anymore 😄
